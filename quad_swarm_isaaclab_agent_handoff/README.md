# Quad swarm Isaac Lab agent handoff pack

Files in this pack:
- `00_INITIAL_INSTRUCTION.md` - first message to the implementation agent
- `01_DECISIONS_AND_SCOPE.md` - required architectural decisions and boundaries
- `02_REPO_SPLIT_AND_TARGET_TREE.md` - exact split between `cpsquare-lab` and the main workspace
- `03_PHASED_IMPLEMENTATION_PLAN.md` - ordered execution plan
- `04_ENVIRONMENT_SPEC.md` - environment and observation/reward spec
- `05_MODEL_AND_TRAINING_SPEC.md` - model and trainer plan
- `06_VALIDATION_AND_ACCEPTANCE.md` - tests, validation steps, and definition of done
- `07_OPEN_QUESTIONS_AND_RISKS.md` - key blockers and mitigation plan
- `08_SOURCES.md` - primary references and useful implementation links

Recommended usage:
1. Give `00_INITIAL_INSTRUCTION.md` to the agent first.
2. Give the remaining files as the implementation brief.
3. Require the agent to keep a parity report as it discovers any differences from the original implementation.
